If the sub has a wiki, the correct size is 1250 x 200.

If it doesn't have a wiki, it's 1250 x 176. If the sub adds a wiki, it won't fit properly anymore.

Get the mod of the sub to use a couple of dots for Page Title and Menu Label.

Set Show in banner menu * to yes and save changes.

BBcode uses Hex Colors or names from this list.

This is incorrect. A sub dread banner without a wiki has a height of 126px, with a wiki it has a height of 176px unless you have to a second row (which you should avoid anyway).

You can achieve this by clicking (CTRL + Shift + M) and then setting a screen width. The height or aspect ratio have nothing todo with this as it is only a fixed width.

There will be also some slight vertical cropping as the .SubBanner class has background-size: cover;
The height is a fixed height of 126px (or 176px) due to the padding of .banner-top (padding: 80px 20px 20px 20px).
Therefore the only way to remain the aspect ratio (or to avoid borders) is to use background-size: cover. This will cause some vertical cropping as everybody has a different screen width while the height remains 126px.

Just inspect element and deactivate background-size: cover; of the .SubBanner class and you will see what I mean. Using some weird image sizes will not influence this at all, as it is a problem due to a fixed height while the width depends on the user's device.

https://developer.mozilla.org/en-US/docs/Web/CSS/background-size

You can read here how background-size: cover; works. And you can try it out yourself by deactivating it of the .SubBanner CSS class.
You will also see that the actual height is either 126px or 176px with one row of wiki entries.

At some point even the correct format (1250px*126px or 1250*176px) will get some cropping. This is because the background size cover will upscale the background image as this CSS property / value is there to avoid empty space and will always fill the div proportionally (so the aspect ratio of the banner will not be stretched).

In fact I can tell you when "at some point" is. It happens at 1521px, because then the div holding the banner as background image gets bigger than 1250px. However, this only applies to you if you use the correct format (1250px*126px or 1250*176px).

The incorrect format suggested here has this unpleasant cropping / upscaling earlier in fact, here it happens at 1374px.
You can try it out and see yourself by clicking (CTRL + Shift + M).
