privacyguides: Rooting Android phones can decrease security significantly as it weakens the complete Android security model. This can decrease privacy should there be an exploit that is assisted by the decreased security. Common rooting methods involve directly tampering with the boot partition, making it impossible to perform successful Verified Boot. Apps that require root will also modify the system partition, meaning that Verified Boot would have to remain disabled. Having root exposed directly in the user interface also increases the attack surface of your device and may assist in privilege escalation vulnerabilities and SELinux policy bypasses.

Content blockers which modify the hosts file (AdAway) and firewalls (AFWall+) which require root access persistently are dangerous and should not be used. AFW+ works based on the packet filtering approach and may be bypassable in some situations.

Hello /u/Beelzebub.I have more information than most on phone security as I have previously had access to a 9 Eyes European police force phone forensics division login for a very large provider. I can provide screenshots of logged in on request.

Use a Privacy-Respecting ROM – If your device supports it, consider de-Googled Android OS like:


GrapheneOS (Best security, but only for Pixel devices)
CalyxOS (Balanced privacy/security)
LineageOS (More devices supported, but less hardened)

GrapheneOS is by far the best operating system and the only one to consider to use. The used Pixel 6a purchased for cash will be more than adequate.
CalyxOS is very far behind. Here are things you lose out on with it:
No duress PIN code. Destruction of evidence will likely be provable by the forensics team but if what is stored on the phone is worse than those charges,
it is much worth it.
ESIM activation like what is used for some paid in Monero SIM plans is handled via the Google eUICC, but Graphene protects against data sharing by this.
The SUPL used for Assisted GNSS is Google.
The connectivity check, DNS check and captive portal check is all provided by the Google. Network time is also handled by Google and only editable by the ADB.
Its hardware attestation provisioning is done by Google unlike Graphene.
MicroG is much inferior to the sandboxed Play used by Graphene.
Only per network MAC random set, but on Graphene this can be done per connection.
Qualcomm XTRA data collection is allowed for Qualcomm devices on the Calyx with only the serial number scrubbed but rest of informations sent.
Some device identifier leaks are solved on Graphene but remain for Calyx.
For Calyx there is no verified boot for system app updates.
There is no possibility for hardware-based security verification.
There is no hardened malloc, secure application spawning, hardened kernel, libc or hardened SELinux. For more facts on the extra hardening it is on their web page.
Calyx cannot disable USB and pogo pin/test pin data, very useful for countering the forensics company.
Calyx auto reboot timer sometimes does not properly engage the before first unlock mode making forensic extraction easier.
As for Lineage, it should not be taken seriously for a secure/private phone setup. It is even less de blobbed.
Graphene is the only device that has proven difficult for the forensics companies. This is not including specialist criminal only crypto phones that you cannot get unless you know somebody who knows somebody. These are mostly built on the Graphene codebase too.
Use a Hardened Web Browser


Brave, Mull, or Firefox with privacy extensions.
Enable uBlock Origin, HTTPS Everywhere, and Disable WebRTC to prevent IP leaks.


Nothing wrong here. Disabling WebRTC is not relevant if you correctly have the whole device Tor set up which I will explain later.

Ditch Google Services


Replace Google Play Store with F-Droid or Aurora Store (anonymous Play Store access).
Use open-source alternatives:
Maps → Organic Maps or OsmAnd
Email → ProtonMail or Tutanota
Messages → Signal (encrypted) instead of SMS

Why would Google Play be installed in the first place? I would not download anything from the Play store without it in a separated profile with the whole device Tor. Please do not recommend these dog shit email providers such as ProtonMail and Tuta. I will not give specific alternative, but they exist. Access Tor only.
Another thing to highlight with the email is that no matter which provider, all can see your emails. The IMAP/SMTP does not provide user or client side
encryption. Any encryption implementation would still allow for the provider to read email. Even if the email is stored encrypted, the mail could still
technically be intercepted whilst being received/sent (SMTP) or whilst being read by the mail client (IMAP). Encrypt all mails with PGP. Download your mail locally and regularly delete it from the server.

Signal is a poor choice unless you are a leftist activist. Briar is better for phone only communications. Avoid the SMS, Snapchat, WhatsApp like it is deadly. Threema codes can be brute forced by forensics company.

Block Trackers and Ads


Install NetGuard (firewall without root) to block app tracking.
Use RethinkDNS (combines firewall, DNS, and tracker blocking).

From NetGuard GitHub page:
"The only way to build a no-root firewall on Android is to use the Android VPN service. Android doesn't allow chaining of VPN services, so you cannot use NetGuard together with other VPN based applications. See also this FAQ."

This makes it useless for the dark net threat model. Always use the whole device Tor, I will explain how in the next section.

Secure Your Network


Always use a No-Logs VPN (e.g., Mullvad, IVPN, ProtonVPN).
Set a Private DNS (e.g., AdGuard DNS or NextDNS).
Use Tor Browser for maximum anonymity (but avoid logging into personal accounts).


Instead of the VPN, the Orbot is a much better option. Install from F-Droid. Configure as whole device in the Graphene or Calyx by going to Settings, Network, pressing Advanced at the bottom, VPN, settings wheel next to Orbot, Always On and Block Connections without.
Tor Browser on Android makes you unique while browsing onion sites, and when paired with the Orbot could cause Tor on Tor situation.


Harden Your Phone Settings


Disable Bluetooth, NFC, and Location when not in use.
Turn off Google Assistant and revoke microphone/camera permissions.
Use Strong PINs & Biometric Locks (avoid face unlock).
Encrypt your phone (enabled by default on Android 10+).

Never enable Bluetooth, NFC or Location. Disable it completely for all non personal profiles on the phone.
Why would the Google Assistant be present on the Calyx or Graphene? This does not make any sense. If this is present on the device you have done something
incorrect.
Never ever ever use a PIN code for your phone. The phone forensics companies such as Greyshift, Cellebrite and MSAB can defeat the ARM based security chips rate limiting already. The RISC based chips are holding out better, but assume they will be broken too. Treat the phone code as if it was an encrypted volume key. At least 25 characters alphanumeric for at least log2() 75 bits of entropy. Never use biometric locks. For Graphene, set a reboot time of two hours or less to bring the phone into BFU state. Always assume a copy of your flash storage can be taken by the forensics team and placed into the cloud to be brute forced. Even if the rate limit chip on the Pixel is defeated it should not be possible to brute force your code.
Encrypt your phone part is redundant. All modern phones that are relevant to this guide have this.

Avoid Sketchy Apps


Uninstall Facebook, TikTok, Temu, and Google Apps (use alternatives).
Check app permissions via Privacy Dashboard (Android 12+).

Why would these applications be installed in the first place?
Instead of checking app permissions, install all suspicious apps in a different Graphene profile.

With Rooting (More Control but Higher Risk)

Rooting gives full control over the system but increases attack surface. Only do this if you understand the risks.

Never ever ever root the secure device! You are aiding the forensics company in their efforts. It is pure stupidity.

Part B

Install a Custom ROM with No Google Services


Use GrapheneOS, LineageOS, or CalyxOS for a clean system.
Install MicroG (if you need Google-dependent apps).

Same comments as other. Sandboxed Play on Graphene is still better than MicroG.

Use AFWall+ (blocks network access per app).
Route all traffic through Tor using Orbot or VPN clients.

I am not familiar with the AFWall. Other is same as other comments.

Remove Preinstalled Bloatware


App Manager or ADB debloater (removes unwanted system apps).

ADB is not requiring root access.

Fake or Randomized Device Data


Use XPrivacyLua (hides IMEI, location, and other identifiers).
MAC Address Spoofing (requires kernel support).

XPrivacyLua does not seem useful given that Graphene already protects these identifiers.
MAC spoofing is already done done by default per network, or better per connection for Graphene.

Never use any Bluetooth gadgets.
Remove microphone, loudspeaker, and cameras.
Use DIN connected microphone and earplugs, so you can decide when the phone can listen.
Consider customizing the battery connection by adding a power switch.
Store the phone in multiple Faraday bags when it's not in use (blocking WiFi, NFR, RFID and 2-5G).

This is mostly solid advice. Removal of small electronics from the phone should be done very carefully and cautiously to avoid breakages.
Faraday bags are not as useful as people think. It is possible to correlate the disappearance and reappearances of the phone from the cell network and certain events. For example, some like to place their personal phones in Faraday bags when using their "activism" phones and vice versa. It is trivial for the cell site analysis to demonstrate the correlation. Instead, leave the personal phone at home and switched on at all times. Another issue is people who use international or Russian SIM cards. The police are not stupid. They can see when 3 Russian SIM cards end up in one location before breaking off and can deduce that you must be all members of a criminal conspiracy.

Having anonymous access to a real computer instead of existing within the contraints of tails and tor unlocks a lot of doors.
